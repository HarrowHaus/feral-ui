# Competitive reassessment — v0.4

## Target set

- neobrutalism.dev: shadcn-style neobrutal component parity and recognizable category ownership.
- 8bitcn: retro shadcn registry, docs, components, blocks, themes, sponsors/community surface.
- pxlkit: deeper ecosystem play: UI kit, icons, templates, visual builder, tests, toasts, monorepo depth.

## Current feral/ui position

v0.4 is no longer a single homepage skin. It has a real React/Vite dogfood app, a component inventory, Radix/cmdk adapters, blocks, template previews, docs atoms, registry generation scripts, and local registry JSON output.

## Where feral/ui is strongest

1. **Original category thesis** — Controlled Variance UI is not just another retro skin.
2. **Voice** — The brand can be funnier than rivals without lying about capability.
3. **Dogfood pressure** — The site imports its own source components.
4. **Registry honesty** — v0.4 generates local item files but still refuses public CLI cosplay.

## Where rivals still beat feral/ui

1. **neobrutalism.dev** has public mindshare and a component docs surface people already understand.
2. **8bitcn** has a trusted registry story, blocks, themes, sponsors/community, and a clearer public release posture.
3. **pxlkit** has ecosystem depth: icons, builder, tests, templates, toast system, packages, and a richer monorepo.

## Hard gaps remaining

- Fresh-app registry install tests.
- Route-backed docs instead of a single-page docs shell.
- Better hard components: sortable/filterable data table, robust date picker, chart adapter, true resizable panels, carousel controls, toast provider.
- A feral icon/ornament pack or builder-like style editor. Pxlkit's builder is a category moat; feral/ui needs its own moat, not a copy.
- Accessibility test script and keyboard walkthrough docs.
- Public-ready license, contributing guide, release notes, and screenshot gallery.

## Judgment

v0.4 is competitive as a **source-library foundation** and still behind as a **public ecosystem**. The next pass should be proof infrastructure: fresh install, tests, docs routing, and a first unique moat.
