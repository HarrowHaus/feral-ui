# v0.8 Execution Unit

## Goal

Move from a working React/Vite preview into a competitive public product surface.

v0.8 is not a component-count sprint. It is a trust and preview-depth pass.

## Build targets

1. Deepen the eight template previews.
2. Polish docs routes and component pages.
3. Add public registry-readiness checks without claiming hosted install support.
4. Keep GitHub Pages deploying the real React/Vite app from dist.
5. Prevent static fallback pages from becoming the product preview.

## Template depth pass

Each template route should become a real multi-section preview:

- SaaS landing
- Dashboard app
- Docs site
- Blog / magazine
- Ecommerce product site
- Agency / portfolio
- AI chat app
- Changelog / updates page

Each template needs:

- hero/top surface
- navigation/app shell
- 3 to 6 real sections
- mobile-safe layout
- source/dependency notes
- theme variation hook

No hero-plus-three-cards placeholder.

## Docs polish pass

Each component page should include:

- live preview
- usage snippet
- source path
- dependency notes
- accessibility receipt
- related components
- variant notes

The docs voice can be rude. The technical claims cannot be fake.

## Registry-readiness pass

Required commands:

```bash
npm run build
npm run registry:build
npm run smoke:registry
npm run smoke:fresh-app
```

The registry is still local until hosted URL smoke tests pass.

## Pages deployment law

GitHub Pages must deploy the real Vite build from:

```txt
./dist
```

The static pages fallback may exist only as emergency documentation. It is not the product preview.

## Acceptance gate

v0.8 is complete when:

- GitHub Pages shows the real React app.
- all template routes render meaningful pages.
- docs pages feel like a library, not a landing-page scroll.
- registry smoke checks pass locally.
- README and roadmap match the current status.
